INVESTOR / PARTNER REPORT
빌드업 4.0 — IP 커머스 인프라 & 공급사 인큐베이팅(AC) 엔진
IP Operator → Pocket Network → Verified Sales → Equity Upside
Infrastructure Equity-for-Service (EFS) Verified Sales OS
1Executive Summary

빌드업 4.0은 '콘텐츠/커뮤니티(IP)'를 거래 엔진으로 바꾸는 커머스 운영 OS입니다. 단순 판매 대행이 아니라, IP 운영(포켓)과 공급사 인큐베이팅을 결합해 매출 수익 + 지분 수익을 동시에 회수합니다.

핵심 가치 제안 (VC 관점)
  • Verified Sales: 체험·추천이 아닌 실매출/재구매 데이터로 검증되는 성장
  • Pocket Network: IP 운영자가 보유한 "포켓(회원)"이 연결되어 자산처럼 누적
  • EFS(Equity-for-Service): 공급사에 OS·마케팅·유통을 제공하고 지분을 선취
  • AC 기능 내장: 공급사는 상품 경쟁력에 집중, 판매/운영은 빌드업이 견인
3년 내 목표(요약)
  • IP 200개 × IP당 연매출 5억 = 연 1,000억 (파생사업 제외)
  • 포켓(회원) 250,000 × 연 4만원 = 1,000억 기여
  • 영업이익률 목표: 10%+ (플랫폼 레버리지 구간 진입)
2Problem & Context
2.1 기존 커머스/플랫폼의 한계
  • 트래픽 의존 → 비용 상승(CAC) → 이익률 하락
  • 링크 기반 판매 → 데이터/고객 자산이 축적되지 않음
  • 공급사 경쟁력 확보가 어려워 상품 소싱의 비대칭 발생
2.2 공급사 관점
  • 유통/마케팅/정산/CS의 운영 부담
  • 확장하려면 결국 법인화·자본 구조화가 필요
2.3 해결이 필요한 방향
  • IP(커뮤니티) 단위로 거래를 생성하고, 포켓(회원)을 자산화
  • 실매출/재구매 기반으로 성과 검증
  • 공급사를 인큐베이팅하여 상품 경쟁력과 기업가치를 함께 상승
  • 성과는 매출 수익 + 지분 가치로 회수

* 빌드업은 "플랫폼이 커지는 회사"가 아니라 "IP와 공급사가 자산을 쌓는 구조"를 제공합니다.

3Solution Architecture
  • IP 운영 구조: 크리에이터는 단순 셀러가 아니라 IP 운영자로 전환 (각자 포켓/몰 보유)
  • Pocket Network: 포켓(상점/회원)이 연결되어 상품·콘텐츠·수익이 이동
  • Cost Plus+ 가격체계: 공급원가 기반 + IP 기여/운영/플랫폼 기여를 반영한 포켓 프라이스
  • Verified Sales Dashboard: 실매출/재구매/구매빈도/회원별 성과를 집계
  • EFS(Equity-for-Service): 운영·마케팅·유통 제공 대가로 지분 선취
운영 프로세스 (요약)
  1. 공급사 발굴 → 상품 경쟁력 검증 → 법인/운영 구조화
  2. IP(포켓) 설계 → 가격/수익 배분 정책 설정
  3. 연결 확산 → 실매출/재구매 데이터로 검증
  4. 성과 기반으로 IP·공급사 확장, 지분 가치 상승

* “관측지표(월 평균 구매)”와 “매출(기간/제품/채널 기준)”은 산정 기준이 다를 수 있어, IR에서는 두 값을 분리해 제시합니다.

5Case Study — Equity-for-Service (EFS)

빌드업은 일부 공급사에 대해 '현금 대신 지분'을 받고 운영·마케팅·유통을 총괄하는 Equity-Based Operating Partnership을 실행합니다.

목포직송 성우농수산 (예시)
  • 법인 설립: 2026.01.21 등기
  • 빌드업 지분: 19% (솔루션 제공 대가)
  • 역할 분담: 공급사는 경쟁우위 상품 소싱, 빌드업은 국내외 판매/운영 견인
  • 목표: 전복·새우 시그니처 상품 설 명절 런칭
AC(인큐베이팅) 논리
  • 공급사 성장 → 기업가치 상승
  • 빌드업은 매출 수익 + 지분 가치 동시 확보
  • 향후 M&A / 자회사 편입 옵션까지 내장
  • 판매 데이터가 누적될수록 검증이 빨라져 확장 비용↓
EFS 포트폴리오 — 지분가치(보수/기준/공격) 시나리오

EFS는 “수수료형 매출”과 별개로, 공급사 기업가치 상승을 통해 옵션 가치를 누적합니다. 아래는 포트폴리오 성장을 기준으로 한 추정치입니다.

빌드업은 ‘거래 수수료’로 현금흐름을 만들고, EFS 포트폴리오를 ‘지분 가치’로 키워 M&A/지분매각/IPO 트랙에서 다중 회수(dual return)를 실행하는 커머스 기반 AC입니다.

회수 시나리오 (예시)
  • M&A: 성장·검증된 공급사를 빌드업 자회사로 편입하여 밸류체인 내재화
  • 지분 매각: 기업가치 상승 구간에서 전략적 투자자/재무적 투자자에게 일부 회수
  • IPO/프로젝트 파이낸싱: 카테고리 리더 공급사의 상장·금융 구조화 트랙 지원

* 추정치이며, 계약 구조·평가 방식·회수 시점에 따라 변동 가능합니다.

* 위 수치는 '모델 설명을 위한 구조 요약'이며, 세부 계약 조건은 케이스별로 상이할 수 있습니다.

6Market & Positioning
6.1 시장 동향
  • 크리에이터의 진화: 광고자 → 셀러 → IP 운영자
  • 플랫폼 종속에서 벗어나 "자산을 쌓는 구조"로 이동
  • AI 자동화로 1인 1몰 운영의 비용이 급격히 하락
6.2 포지셔닝
  • 단순 커머스 툴이 아닌 IP 커머스 인프라 레이어
  • 수수료 모델 + 지분 기반 업사이드
  • 진입장벽은 기술이 아니라 연결된 관계망/데이터
7Unit Economics & Business Model
7.1 수익원
  • 거래 수수료/저작료 (포켓 간 연결 거래 포함)
  • IP 운영 솔루션 (구독/패키지)
  • 데이터 리포트 / API / 운영 도구(SaaS)
  • EFS 지분 가치: 인큐베이팅 기업가치 상승분
7.2 유닛 이코노믹스 (핵심 가정)
  • 확장의 단위: IP 수 × 포켓(회원) 수
  • 포켓당 연 매출: 4만원 (보수적 가정)
  • IP당 연 매출: 5억 (2028 목표치)
  • 운영 효율: 자동화로 CAC↓, 반복 운영비↓
8Roadmap & Milestones
  1. Phase 1 (0~3개월): 핵심 IP 운영자 확보, 표준 온보딩/가격정책 템플릿화
  2. Phase 2 (4~12개월): IP 클러스터 확장(푸드·뷰티 등), 공급사 인큐베이팅 파이프라인 구축
  3. Phase 3 (13~24개월): IP 200개 목표 트랙 진입, 해외(일본/몽골/필리핀 등) PoC
9Risks & Mitigation
9.1 주요 리스크
  • IP 확장 속도 저하(운영자 영입)
  • 공급사 품질/재고/CS 병목
  • EFS 계약의 표준화/법무 프로세스
9.2 대응 전략
  • 탑티어 IP 운영자 중심 클러스터 전략
  • 표준 온보딩 킷·정산/CS 템플릿·데이터 대시보드
  • EFS 표준 계약/실사 체크리스트로 리스크 관리
10KPI Dashboard (Targets)

  • EFS Upside: 지분가치 시나리오(보수/기준/공격)로 추가 회수 옵션 내장
해외 확장 (2027~)

  • 시스템은 유지, 언어/카테고리/파트너만 로컬라이징
11Investment / Partnership Ask

  • 투자: IP 확장/해외 PoC/EFS 포트폴리오 확장(법무·실사·운영 표준화 포함)
  • 공동 사업: 데이터 기반 상품개발, 공동 브랜드, M&A/IPO 트랙 협업

Appendix — 핵심 메시지

빌드업은 IP 운영자(포켓)와 공급사 인큐베이팅을 결합해 "매출 수익 + 지분 수익"을 동시에 회수하는 커머스 인프라입니다. 확장의 단위는 트래픽이 아니라 IP 수 × 포켓(회원) 수이며, 실매출/재구매 데이터로 성장의 재현성을 증명합니다.
